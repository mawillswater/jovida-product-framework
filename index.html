🧭 教练视角闭环

Jovida 的宏观与微观教练闭环

把"教练做的事"还原成两层闭环：宏观闭环管节奏（了解 + 诊断 + 制定策略 → 持续陪练 → 复盘 → 调策略），微观闭环管每一次单点干预（观察 → 触发 → 干预 + 辅助 → 行为）。微观闭环完全嵌套在宏观闭环的"持续陪练"节点里——这是产品里所有功能的归位坐标。

两层闭环 · 互相嵌套

Jovida 的宏观与微观教练闭环

宏观闭环决定"现在该练什么"——用户的目标、现状、问题诊断、本阶段的 Action 编排和资源池。这套策略被注入"持续陪练"节点，成为微观闭环每天迭代的素材库。一次微观循环约几分钟，宏观循环约一段时间。日常 99% 的产品体验在微观闭环里发生，但一旦微观跑偏了，宏观闭环会通过复盘把它拉回来。

宏观闭环 · 周 / 季

了解 + 诊断 + 制定策略 → 持续陪练 → 复盘 → 调策略

宏观闭环关键时刻

4 个不能糊的瞬间

① 第一次打开 开放对话弄清楚用户是谁、想去哪儿，输入什么都接（文字、链接、图、视频）。当晚就得给出第一份计划，否则会被当成"又一个空泛的 AI 助手"。
② 目标增减 陪练过程中用户随时能加 / 删目标。每次变化都要重新对齐节奏，不能只塞进列表就完事。
③ 周末复盘 陪用户回头看一周做了什么、走偏在哪儿。给出下一段方向草稿——可接受、可手动改、可让 Agent 重写。
④ 季度切换 每 12 周一次大复盘——目标可能整体换一批。这是宏观闭环的"重启感"，而不是失败感。
⚡ 教练微观闭环

每一次"被看见"都是一次完整的小循环

一个好教练每天会盯着你做几十次小动作：知道你刚做了什么 → 在最对的时刻提醒你 → 给你最现成的支持 → 看你完成 → 再回到下一轮。Jovida 把这件事拆成 4 个明确的环节。

1

观察 · Agent 看用户

教练不能闭着眼陪练。Jovida 通过两条通道持续了解用户的微观行为和状态——一条是用户主动告诉 Agent（拍午餐、传 crush 聊天截图、写一段感受），另一条是 Agent 在后台默默收集（Apple Health 步数和睡眠、地理位置、当地天气、屏幕使用时间、关注对象的公开动态）。

微观闭环关键决策

4 个不能糊的瞬间

① 上传摩擦 主动入口 ≤ 2 步可达，被动信号默认默认开启。多一步就丢一半信号——这是闭环的源头水池。
② 触发时机 不是定时推，是"刚刚发生了什么"。窗口错过宁可静默，下一个时段再来。
③ 干预成本 Action 一步可完成，资源全部预挂，不让用户跳出去找。Nudges 不强求完成。
④ 反馈关联 不要泛泛"恭喜"。要说"你 collaborate 答错了，混淆了协作和采集"——证明 Agent 真在记。

📦 产品的基本单元

4 个基本概念，撑起整个产品

Jovida 里所有功能都建立在 4 个基本概念之上：Goals（用户的目标）→ Actions（要完成的动作，5 类）→ Resources（Agent 给的辅助支持）→ Streak（奖励落地）。理解这 4 个概念就能把产品里任何一个屏幕归位。

1

Goal

Goal 是用户的目标 / 心愿。每个用户可以同时拥有多个 goals——通过 Onboarding 创建第一批，之后随时可以增减。每个 goal 都关联一组 actions 和 resources。

3

Resources

Resource 是 Agent 给用户的辅助支持——通过 nudge 形式呈现，用户可以选择消费但不强制。这是 Action 的"配套补给"，用来降低完成 action 的门槛。

  • ①形式：商品 / 餐厅 / 教程 / 音频 / 文章 / 数据卡 / nudge 提醒
  • ②不强求：不消费不影响 streak，只是把"现成的支持"摆到面前

Streak

Streak 是用户每天的奖励落地。每天至少完成一次 action 即获得 streak。请假也算 streak——这是 Jovida 区别于多邻国式催促的关键设计。连续 streak 累积可以解锁特殊里程碑（30 / 100 / 365 天）。

  • ①触发：每天完成至少 1 个 action 即 +1 streak。完成多个不会叠加，只算 1。
  • ②挽回：断胜当天完成 3 个 action 可补回连胜（C1 路径）
  • ③请假：用户主动请假 / 系统检测特殊状态时，休息日也算 streak（C3 路径）
  • ④里程碑：30 / 100 / 365 天解锁限定徽章 + 全屏特殊动效，是产品的对外传播契机

🎭 角色与关系

不只是 Agent，是有人格的同伴

角色与关系是提升用户外部动机和情绪价值的关键手段。一个有人设、能记住你、随你成长的角色，比"高效完成任务"更能让人留下来。类比多邻国（Duolingo Duo）——sassy 的小猫头鹰、装死、威胁式提醒、成为文化梗，把"语言学习 App"变成"和 Duo 的人际关系"。

🦉

Duolingo Duo 角色系统的关键要素

① 视觉 IP 一致性（绿猫头鹰跨场景识别） · ② 性格人设（sassy / passive-aggressive，不是礼貌助手） · ③ 情绪状态反应（你不练 → 难过 / 威胁 / 装死） · ④ 多角色 cast · ⑤ 关系记忆（引用你的 streak / 错题 / 习惯） · ⑥ 跨平台延展（TikTok / Twitter / 表情包） · ⑦ 表情贴纸系统（动画 + emoji 表达情绪） · ⑧ Lore / 故事（Duo 和 Lily 的恋爱）。下面 4 个维度抽取最有产品意义的部分应用到 Jovida。

1

角色人设 / 性格

不同性格档位的角色给用户不同的情绪供给。关键不是"找一个完美角色"，而是给用户选择权——sassy 的不一定适合所有人，温柔的也未必能驱动所有人。提供 4-5 种典型档位让用户选适合自己的"陪伴味道"。

典型档位（暴躁 ←→ 温柔 维度）：

  • 💣暴躁炸弹人——戏剧化、夸张威胁、Duo 那种 sass。例："你又没做？我快炸了！"
  • 🤗温柔支持者——闺蜜风、永远站在你这边、不批评。例："没事，慢慢来。"
  • 🥋严厉教练——军人风、不废话、推 stretch。例："别废话，再来一组。"
  • 🪷佛系陪伴——低压、不评价、做不做都行。例："做不做都好，听你的。"

关键设计点：性格不是"礼貌客气"——必须有边。Duo 之所以红是因为它敢威胁、敢示弱、敢哭、敢嘲讽。礼貌助手只会被遗忘。

3

关系记忆与演进

角色不是无状态的回复机器——它记得你。从 day 1 的陌生客气，到 day 30 的默契，到 day 100 的挚友，关系会变深。每次对话都引用"你之前说过的"，让用户觉得"它在和我成长"。

关系深度 4 阶段：

  • ①陌生（Day 1-7）——角色礼貌、保守、用敬语，主要是"了解你"
  • ②熟悉（Day 7-30）——开始用昵称、记得偏好、敢提建议
  • ③默契（Day 30-100）——敢开玩笑、有 inside jokes、知道你的痛点和高峰
  • ④挚友（Day 100+）——像老朋友，引用半年前的事，有"我们的故事"

记忆类型：① 用户的偏好 / 痛点（"你最讨厌跑步"）；② 重要时刻（"你 day 7 第一次破纪录"）；③ 反复犯错（"retain 这词错了 5 次"）；④ 情绪状态（"上周你说累了，我没敢推 stretch"）。这些细节是 Duo 真正粘人的原因。

视觉表情系统（跨场景情绪表达）

表情系统的 3 个层次：

  • ①基础情绪库——20-30 个高频表情（开心 / 嘲讽 / 装死 / 撒花 / 流泪 / 期待 / 怒 / 困 / 倒计时...）
